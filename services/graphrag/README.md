# GraphRAG Service

The `graphrag` service is the retrieval engine for the Knowledge Graph. It utilizes a **LlamaIndex** agent to navigate and query the data stored in the graph database.

## Graph Generation Pipeline

The data consumed by this service is generated by the **GraphGen** pipeline. The generation process transforms unstructured input into a structured, semantic, and navigable graph through the following stages:

### 1. Lexical Graph Construction
Input data (e.g., life logs, transcripts) is parsed into a temporal hierarchy to preserve chronological context:
- **Structure:** `Day` → `Segment` → `Episode` → `Chunk`
- **Purpose:** Anchors every extracted fact to a specific point in time and source document.

### 2. Entity & Relation Extraction
The pipeline processes text chunks using a hybrid approach:
- **NER:** Uses **GLiNER** or **Spacy** to identify high-confidence entity candidates.
- **LLM Extraction:** Uses **Groq-hosted Llama 3** models to extract semantic entities and relationships, filtering for relevance and quality.

### 3. Semantic Enrichment
- **Embeddings:** Vector embeddings are generated for all text chunks and entities using `all-MiniLM-L6-v2`.
- **Resolution:** Entities are resolved and deduplicated based on vector cosine similarity and name matching.

### 4. Community Detection (Hierarchical)
To support high-level summarization and "global" queries:
- **Clustering:** The **Leiden algorithm** is applied to the graph to detect communities of densely connected entities.
- **Summarization:** LLMs generate concise summaries for each identified community, creating `Topic` and `Subtopic` nodes.

### 5. Hybrid Storage Architecture
The final graph is persisted in a dual-database setup:
- **FalkorDB:** Stores the graph topology (Nodes, Edges, Properties) for traversal.
- **PostgreSQL (pgvector):** Stores 384-dimensional vector embeddings for fast similarity search and hybrid retrieval.
